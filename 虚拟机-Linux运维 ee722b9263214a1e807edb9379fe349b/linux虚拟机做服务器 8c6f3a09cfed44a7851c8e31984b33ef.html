linux虚拟机做服务器

端口转发

【如何实现：主机用公网IP下，虚拟机可以被访问到】

可以使用网络地址转换（Network Address Translation，NAT）或端口转发（Port Forwarding）的方法。

  1. 使用NAT：这种方法适用于您的主机与虚拟机之间是私有网络（例如，使用NAT模式的虚拟机）的情况。
    • 首先，确保主机与虚拟机处于同一网络，并且虚拟机已经获得了内部网络的IP地址。
    • 在主机上，启用IP转发功能（Forwarding）：
      sudo sysctl -w net.ipv4.ip_forward=1
    • 在主机上设置NAT规则，将外部流量转发到虚拟机：
      sudo iptables -t nat -A PREROUTING -p tcp --dport <外部端口> -j DNAT --to-destination <虚拟机IP地址:端口>

      上述命令将外部端口的TCP流量转发到虚拟机的特定IP地址和端口。

    • 如果您的主机上有防火墙（比如iptables），确保允许通过转发的流量：
      sudo iptables -A FORWARD -p tcp --dport <虚拟机端口> -d <虚拟机IP地址> -j ACCEPT

      上述命令将允许转发到虚拟机的TCP流量通过防火墙。

    • 现在，外部网络可以通过访问主机的公共IP地址和相应的端口来访问您的虚拟机。
  1. 使用端口转发：这种方法适用于您的主机与虚拟机之间的连接方式是桥接（Bridge）或主机模式（Host-only）的情况。
    • 首先，确保主机与虚拟机处于同一内网，并且虚拟机已经获得了内部网络的IP地址。
    • 在主机上（假设为windows）查看端口转发设置：
      netsh interface portproxy show all

      设置端口转发规则：

      netsh interface portproxy add v4tov4 listenaddress=0.0.0.0 listenport=<本地端口> connectaddress=<目标IP地址> connectport=<目标端口>
    • 检查新添加的端口转发规则是否正确：
      netsh interface portproxy show all

防火墙开放端口

使用ufw工具进行简单的端口管理：

#检查 ufw 的状态：
sudo ufw status
#允许单个端口，例如：
sudo ufw allow 443
#允许多个端口，例如：
sudo ufw allow 8000,8010,8020,7000,7010,25,143,993,995,587,110,465/tcp
#启用 ufw：
sudo ufw enable
#如果需要，可以查看和管理其他规则，例如删除规则：
sudo ufw delete <rule_number>
