1、存储空间不足

【这时候会有以下表现】

1、下载东西或者创建目录&文件的时候报错：/var/cache/apt/archives/空间不足

2、普通用户登不上，root登的上

【解决方式如下】

  • 1、删除大文件（有些文件不好找到，不推荐）

    （1）用以下命令来查看存储空间占用情况：

    $ df -h
    • 回显分析
      文件系统                       大小  已用  可用 已用% 挂载点
udev                           1.9G     0  1.9G    0% /dev
tmpfs                          390M  1.3M  389M    1% /run
/dev/mapper/kali4pwn--vg-root   15G   15G     0  100% /    #可以看到，这里是被占满了！
tmpfs                          2.0G     0  2.0G    0% /dev/shm
tmpfs                          5.0M     0  5.0M    0% /run/lock
/dev/mapper/kali4pwn--vg-home   13G  906M   12G    8% /home
/dev/sda1                      455M   84M  347M   20% /boot
tmpfs                          390M   96K  390M    1% /run/user/1000

    （2）查看当前目录下所有>50M的文件和目录的位置+大小：

    du -ah ./ | awk '$1 ~ /[0-9]M/ && $1+0 > 50 {print}' | sort -hr
    • 这个命令的工作原理如下
      • du -h ./：显示当前目录及其子目录的文件大小。
      • awk '$1 ~ /[0-9]M/ && $1+0 > 50 {print}'：使用 awk 命令过滤出大于50M的结果。$1 ~ /[0-9]M/ 是一个正则表达式，用于匹配以M为单位的文件大小。$1+0 > 50 用于过滤出大于50的结果。
      • sort -hr：使用 sort 命令将结果按照文件大小从大到小排序。h 选项表示以人类可读的数字大小（例如，1K、234M、2G）排序，r 选项表示反向排序（即从大到小排序）。

    （3）删除一些日志文件&临时文件：

    使用命令du -sh /var/log/查看日志文件目录的大小，并用rm命令删除不需要的日志文件

    使用命令du -sh /tmp/查看临时文件目录的大小，并用rm命令删除不需要的临时文件

    （4）删容器+镜像：

    docker rmi -f $(docker images -qa)
docker rm -f $(docker ps -a -q)

2、防火墙导致网络传输失败

在虚拟机与宿主机的telnet连接建立过程中遇到的问题
  • 一些网络连接失败可能是因为防火墙问题引起的：
    sudo systemctl status firewalld.service
sudo systemctl stop firewalld.service
sudo systemctl disable firewalld.service

  • 也有可能是SELinux设置了连接的权限引起的：

    修改/etc/selinux/config文件中的SELINUX=”” 为 disabled，然后重启

3、yum无法安装某些系统需要的软件

centos7的yum比较垃圾，有时候镜像站里明明有某个软件，但他就是下载不下来，这时候就需要手动下载安装：

例如：

wget http://mirror.centos.org/centos/7/extras/x86_64/Packages/fuse-overlayfs-0.7.2-6.el7_8.x86_64.rpm
rpm -ivh fuse-overlayfs-0.7.2-6.el7_8.x86_64.rpm

其中，http://mirror.centos.org/centos/7/extras/x86_64/Packages/这个网站可以存一下，要下载什么包的时候就可以在这里面找

4、因为墙导致github下载不了东西

有3种解决思路，一种是在虚拟机中开梯子，另一种是在宿主机中开梯子下载zip文件，解压后用xftp传给虚拟机，还有一种是使用github的镜像站

5、权限不足

chmod a+rwx 文件名 #为某个文件添加rwx权限

chmod a+rwx -R 目录路径 #为某个目录下的所有文件添加rwx权限

su #以su来运行shell，如果有些软件当时下载时是root，那可能普通用户运行不了他，建议是所有软件都用root下载

6、xftp无法连接

可能是虚拟机没有连接上虚拟网卡&虚拟网桥，可以尝试用浏览器访问百度，如果连不上，尝试重启虚拟机有时候可以重连上虚拟设备。

1、查看当前ip地址：

有可能是ip地址对应错了

用这个命令查看ip地址：

$ ifconfig
......
eth0: flags=4163<UP,BROADCAST,RUNNING,MULTICAST>  mtu 1500
        inet 192.168.11.133  netmask 255.255.255.0  broadcast 192.168.11.255
......

可以看到局域网ip的192.168.11.133，再看看xftp连接的ip是不是这个：

2、查看22端口有没有被别的进程占用

xftp使用ssh，ssh默认是22端口

用这个命令查看ssh进程占用了哪个端口

netstat -tuln | grep sshd

如果没有回显，可能是两种情况：

（1）22端口被占用

（2）ssh进程没有启动

3、重启ssh服务

也可能是ssh服务没有开启

用这个方式开启：

$ sudo apt install openssh-server
$ sudo systemctl restart ssh #重启ssh
