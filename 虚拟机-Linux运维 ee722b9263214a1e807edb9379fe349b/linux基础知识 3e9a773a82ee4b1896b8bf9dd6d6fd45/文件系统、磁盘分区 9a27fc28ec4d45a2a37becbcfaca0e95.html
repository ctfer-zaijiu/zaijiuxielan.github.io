文件系统、磁盘分区

一、相关基本概念

1-文件系统结构

一个电脑可以装多个磁盘，一个磁盘可以分四个分区且装一个文件系统。

所以就是例如：

/dev/sda1：磁盘1的分区1
/dev/sda2：磁盘1的分区2
/dev/sda3：磁盘1的分区3
/dev/sda4：磁盘1的分区4
#------------------------
/dev/sdb1：磁盘2的分区1
/dev/sdb2：磁盘2的分区2
#------------------------
/dev/sr0：光驱设备

2-文件系统与分区方式

文件系统：

【定义】

作用于硬盘用来管理文件的系统

其功能是：保存文件的数据+属性

【结构】

其中，文件系统会将文件的数据和属性分开存储：

文件数据保存在data block（即数据块）中，块的大小和数量在格式化后就已经确定不能更改，文件大小以块为单位对齐，也就是不能把两个文件放在同一个块里

文件属性则保存在inode（即i节点）中，记录了文件的创建修改时间，所有者，读写权限等（不含文件名）

此外还有一个super block用来记录整个文件系统的信息，如块的总数，已经使用的块的个数

注意，目录也用data block描述，其中存放了该目录下一级的所有子目录和子文件的文件名

【文件系统碎片整理】

其原理是把一个大文件的几个块整理到物理地址相邻的地方，从而实现提升其读写速度。

【常见文件系统】

XFS ：

是一种高性能的文件系统，最初由 SGI 开发，现在由 Linux 社区维护。它在处理大文件和大分区时表现出色。适用于大型数据存储和高性能服务器环境。

ext系列：

从ext到ext2到ext3到ext4一脉相承，其中最新版的ext4引入了许多新特性，包括更快的文件系统检查、更大的文件和分区支持、更快的写入性能、更好的文件碎片整理等。

适用于通用用途的计算机，是目前比较主流的linux文件系统。

sda、sdb磁盘：

linux用/dev/sda来标识其接入的第一个磁盘驱动，用/dev/sdb来标识其接入的第二个磁盘驱动，以此类推

其中，sd表示“SCSI或SATA硬盘”，如果是hd则表示IDE硬盘

sda1、sda2分区：

一个磁盘可以分4个大分区，表示方式例如：/dev/sda1、/dev/sda2、/dev/sda3、/dev/sda4

这其中可以把这四个分区都作为主分区，也可以分三个主分区一个做扩展分区

然后可以再在扩展分区里分出多个逻辑分区，表示为例如：sda5、sda6、sda7…

sr0、cdrom附加设备盘：

linux用/dev/sr0来表示光驱设备（通常是DVD或CD-ROM驱动器）

然后它会再起一个别名，如/dev/cdrom，然后软链接向/dev/sr0，用这个别名的好处是更方便记忆

【常见分区类型】

/分区：

即根分区，用于挂载/

/boot分区：

用于挂载/boot

存放引导操作系统的文件，用于开机启动

SWAP分区：

将一部分磁盘空间设置为“交换分区”，可以实现“拓展内存大小”的作用

其原理是：

当物理内存不够用时，os会从物理内存中取出一部分用不上的数据先暂时放到交换分区中

一般建议将交换分区的大小设置为物理内存的1~2倍。

/opt分区：

用于挂载/opt

其通常被规划用于安装和管理不包括在操作系统核心或标准软件库中的软件包（即一些附加组件&第三方应用程序）

其中，linux系统必须要有根分区和boot分区才能运行，其他的分区可以不需要

二、磁盘分区

1-定义

磁盘分区独立于linux的文件系统结构，但是可以设置将文件系统的那些目录对应在哪些分区中

对linux系统进行磁盘分区，有以下意义：

1、数据隔离：分区之间数据隔离，提高安全性

2、任务对应：比如有的分区专门用来挂载，有的分区专门用来缓存等等

3、性能优化：不同的分区可以设置不同“簇”大小等配置，以适应其对应的任务

2-结构

一般来说，按照内存从小到大相贴顺延下去，分区的顺序为：

1、boot分区

2、swap分区

3、根分区

3-fdisk工具

【定义】

是一个linux的磁盘分区管理工具

【相关指令】

1、添加硬盘设备时，将其作为一个新分区或划入根分区

2、硬盘阵列都挂载在根分区，此时有一个硬盘损坏了，要求将其地址移出根分区

2、分区的增加、删除、类型修改

（1）进入该磁盘

sudo fdisk /dev/sda

（2）删除现有分区&创建新分区

按d进入删除模式，输入分区号回车删除指定分区

按n进入创建模式，输入分区号+类型选primary+起始地址（有提示）+大小回车创建新分区

（3）修改分区的类型

输入 t 进入类型修改模式，输入分区号修改指定分区的类型，其中：

82为linux swap类型、83为linux文件系统类型，像是根分区、OPT分区用这种类型即可

此处的数字均为16进制，也就是说，82其实就是0x82

（4）保存并退出

输入 w ，以保存并退出fdisk。

3、格式化并挂载这些分区：

  • 建议把OPT分区格式化成ext4类型，指令如下：

    取消挂载：

    sudo umount /dev/sda3

    然后格式化：

    sudo mkfs.ext4 /dev/sda3
  • 至于swap分区则不需要指定类型，直接格式化即可，指令如下：
    sudo mkswap /dev/sda4

    然后需要以下命令激活swap分区：

    sudo swapon /dev/sda4
  • 然后更新/etc/fstab文件以确保分区在启动时能够正确挂载：
    该文件设定了boot时要进行的挂载策略
    su
chmod a+rwx /etc/fstab
vim /etc/fstab

    然后写入类似这样的格式：

    /dev/sda2    /       ext4    defaults 0 1
/dev/sda3    /opt    ext4    defaults 0 2
/dev/sda4    none    swap    sw       0 0
    关于最后三列：
    • defaults：这是用于指定挂载选项的字段，其中 defaults 通常包括一些默认的挂载选项，如读写权限等。
    • 0：这是用于备份目的的字段，通常设置为 0，表示不需要备份。
    • 2：这是用于检查顺序的字段，通常设置为 2，表示在文件系统检查时应该进行检查。

三、挂载

【定义】

用于把一个外部文件系统挂载到到 Linux 文件系统层次结构中，以便访问它们

【命令】

mount [选项] 外部文件系统 挂载点

【选项】

一些常见的选项包括：

-t 文件系统类型：指定要挂载的文件系统的类型，例如 ext4、ntfs、nfs 等。

-o 选项：指定文件系统的挂载选项，例如读写权限、用户权限等。

【示例】 

mount /dev/sdX /mnt/mydisk #将硬盘分区/dev/sdX（这里是刚插进去的u盘）挂载到linux的/mnt/mydisk目录
mount -t nfs 192.168.1.100:/share /mnt/nfsshare #挂载一个共享文件夹
umount /mnt/mydisk：取消挂载
