权限与用户组管理

一、linux的三种特殊权限

这些权限可以在文件或目录上设置，以影响它们的执行或访问方式。

1、SUID（Set User ID）：

  • 标识符： s 或 S
  • 作用：该程序在执行时会以文件所有者的身份而不是执行者的身份运行。
    这对于需要以特殊权限运行的程序非常有用。
  • 示例：
chmod u+s executable_file
# 或者：
chmod 4755 executable_file

2、SGID（Set Group ID）：

  • 标识符： s 或 S
  • 作用： 该目录中新创建的文件会继承“目录的组所有权”。
    而不是继承创建者的组所有权。

    这对于多用户共享目录或项目工作空间很有用。

  • 示例：
chmod g+s directory
# 或者
chmod 2755 directory

3、Sticky Bit：

  • 标识符： t 或 T
  • 作用： 只有“文件所有者和目录所有者”才能删除自己创建的文件。
    通常用于临时目录。
  • 示例：
chmod +t directory
# 或者：
chmod 1757 directory

二、用户、用户组权限概述

1-查看当前系统的所有用户

【指令】

cat /etc/passwd

【回显】

  • 如

    1. 用户名：用户的登录名，如sys、sync等。
    1. 密码：用户密码的占位符。在早期的Unix系统中，这个字段存储的是加密后的用户密码。但出于安全考虑，现代Linux系统通常将加密后的密码存储在/etc/shadow文
    1. 件中，而在/etc/passwd文件中这个位置通常是一个x或者一个星号（*）。
    1. 用户ID (UID)：用户的唯一标识符。系统用户（如sys、mail等）通常有较低的UID（例如0到999），而普通用户的UID通常从1000开始。
    1. 组ID (GID)：用户所属主组的唯一标识符。组信息存储在/etc/group文件中。
    1. 用户全名或描述：有关用户的一些描述信息，也可以是用户的全名。例如，games用户的描述是games。
    1. 家目录：用户登录后的初始工作目录。例如，mail用户的家目录是/var/mail。
    1. 登录Shell：用户登录后所使用的shell程序。如果设置为/usr/sbin/nologin，这意味着用户不能登录系统。这通常用于系统账户&有系统权限的服务账户，以提高安全性。

3-一些安全相关设置操作

检查并删除空密码账号：

awk -F: '($2 == "") {print $1}' /etc/shadow   # 这个命令会输出所有密码字段为空的账户的用户名
passwd 用户名 #为该用户设置密码

设置账号相关配置文件的不可改变位（运维应急）：

chattr +i /etc/passwd  # 系统中所有用户的基本信息，其中x是密码的占位符，表示密码被加密后移到shadow文件中去了
chattr +i /etc/shadow  # 用户的加密密码以及与密码相关的管理信息
chattr +i /etc/group   # 群组相关配置信息
chattr +i /etc/gshadow # 群组的加密密码

锁定某个账户：

当一个用户的密码被锁定时，意味着该用户不能使用其密码来登录系统。密码锁定通常用于管理目的，比如暂时禁止某个用户访问系统，而不是完全删除该用户的账户。

passwd -l accountName

三、设置文件权限

1-基础权限

chmod将当前目录下所有文件改为宽权限

chmod a+rwx -R ./

ls查看某文件权限

ls -al

2-扩展权限

chattr用于修改文件或目录的扩展属性：

# 设置文件的属性为只可追加&取消文件的只可追加属性
chattr +a filename
chattr -a filename

# 设置文件的属性为不可删除(root也不能删除)&取消文件的不可删除设置
chattr +i filename
chattr -i filename

lsattr命令则用于显示文件或目录的这些扩展属性：

lsattr -a
