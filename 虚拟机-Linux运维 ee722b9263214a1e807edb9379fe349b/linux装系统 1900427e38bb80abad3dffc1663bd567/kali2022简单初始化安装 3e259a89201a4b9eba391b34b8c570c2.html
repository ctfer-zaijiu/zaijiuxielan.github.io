1、创建kali虚拟机

1、下载vmware

从官网下载vmware，然后去百度搜索激活码，多试几次就成功了

2、镜像文件下载

建议下载kali2022，地址：http://old.kali.org/kali-images/kali-2022.4/

  • 其中涉及的注意事项

    下载.iso后缀的

    不要下载微缩的内核版，因为那里面会缺少很多像vim、cat这样的工具

    不要下torrent，那个是联网下载安装器，会装很久

    不要下everything版，那个很大很臃肿

    下在D盘自己找得到的地方，最好是全英文路径

    不要把.iso文件删掉

  • 其他操作系统的下载

    centos7，建议从这里下载：centos-7-isos-x86_64阿里源

3、虚拟机创建

直接点开vmware，然后创建虚拟机即可

  • 其中，相关配置选择如下：

    路径：

    iso文件和虚拟机配置文件路径建议都采用全英文！

    操作系统选项：

    kali的话对应的vmware里的操作系统选项是ubantu64位

    配置：

    建议使用：2cpu，1核，4G内存，50G存储空间

    其他地方默认，网络模式NAT和桥接都可以

    用户名密码：

    root和root即可，防止忘记

    有一次帮新生创建虚拟机，他就输了他喜欢的密码，结果自己把密码给忘了

    语言环境：

    kali2022中可以直接选中文

    • 对于其他操作系统的建议

      其他操作系统建议使用中文地区但使用英文作为系统语言，同时输入法使用英文

      因为这可以避免很多因为字符集问题导致的乱码现象

      目前来说，使用ubantu22（或者是ubantu16？忘了。）时，xftp会出现中文乱码现象，kali2022xftp不会中文乱码，kali2019只要系统环境选中文，一进系统就乱码

可以看到要做的事情：

  • 其中，一些关键字的解释
    • deb 开头的行用于指定二进制软件包的源。deb-src 开头的行用于指定源代码软件包的源。
    • main, restricted, universe, 和 multiverse 是软件包组的组成部分，它们代表不同的软件包类别：
      • main：包含Kali官方支持的自由软件。
      • non-free：包含非自由软件，通常包括一些有限支持的软件包。
      • contrib：包含依赖于非自由软件的自由软件包。

  • 一些相关报错的处理：
    • 问题1：报错xxx在xxx中被配置了多次

      报错信息如下：

      W: 目标 Packages (stable/binary-all/Packages) 在 /etc/apt/sources.list.d/archive_uri-https_download_docker_com_linux_ubuntu-kali-rolling.list:1 和 /etc/apt/sources.list.d/archive_uri-https_download_docker_com_linux_ubuntu-kali-rolling.list:3 中被配置了多次
W: 目标 Translations (stable/i18n/Translation-zh_CN) 在 /etc/apt/sources.list.d/archive_uri-https_download_docker_com_linux_ubuntu-kali-rolling.list:1 和 /etc/apt/sources.list.d/archive_uri-https_download_docker_com_linux_ubuntu-kali-rolling.list:3 中被配置了多次
W: 目标 Translations (stable/i18n/Translation-zh) 在 /etc/apt/sources.list.d/archive_uri-https_download_docker_com_linux_ubuntu-kali-rolling.list:1 和 /etc/apt/sources.list.d/archive_uri-https_download_docker_com_linux_ubuntu-kali-rolling.list:3 中被配置了多次
W: 目标 Translations (stable/i18n/Translation-en) 在 /etc/apt/sources.list.d/archive_uri-https_download_docker_com_linux_ubuntu-kali-rolling.list:1 和 /etc/apt/sources.list.d/archive_uri-https_download_docker_com_linux_ubuntu-kali-rolling.list:3 中被配置了多次
W: 目标 Contents-deb (stable/Contents-all) 在 /etc/apt/sources.list.d/archive_uri-https_download_docker_com_linux_ubuntu-kali-rolling.list:1 和 /etc/apt/sources.list.d/archive_uri-https_download_docker_com_linux_ubuntu-kali-rolling.list:3 中被配置了多次

      处理：

      这里好像是docker仓库配置出错

      用vim打开这里提到的那些文件，将相关配置声明内容注释掉

    • 问题2：有时候使用了镜像源后，一些包会装不到

      【分析】

      可能是源里面的内容不全，或者是源最近出问题了

      【解决】

      $ vim /etc/apt/sources.list

      在配置列表里面追加写入

      #常规源：
deb http://http.kali.org/kali kali-rolling main non-free contrib
deb-src http://http.kali.org/kali kali-rolling main non-free contrib
#清华源：
deb https://mirrors.tuna.tsinghua.edu.cn/kali kali-rolling main contrib non-free
deb-src https://mirrors.tuna.tsinghua.edu.cn/kali kali-rolling main contrib non-free

      然后更新缓存

      sudo apt-get update
    • 问题3：仓库xxx没有没有 Release 文件

      报错信息如下：

      E: 仓库 “https://download.docker.com/linux/ubuntu kali-rolling Release” 没有 Release 文件。
N: 无法安全地用该源进行更新，所以默认禁用该源。

      分析：

      尝试从 https://download.docker.com/linux/ubuntu 更新 Docker 时，因为 Kali Linux 使用的是 kali-rolling 版本，而该 URL 并不支持 Kali Linux，导致无法找到相应的 Release 文件。

    • 问题4：没有合法的证书

      解决：手动下一个证书~

      wget -q -O - https://archive.kali.org/archive-key.asc | apt-key add -
