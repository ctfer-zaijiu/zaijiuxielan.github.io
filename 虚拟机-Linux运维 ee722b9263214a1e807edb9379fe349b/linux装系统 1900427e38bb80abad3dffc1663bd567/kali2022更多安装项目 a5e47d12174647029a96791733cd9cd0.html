1、安装python3

  • 以下方法适用于kali和ubantu：

    $ su

    $ apt install python3.10
    $ apt install python3-pip

    如果失败，可以试试：
  • 以下方法适用于centos7：

    1、在开始安装之前我们需要先安装Python基础依赖包：

    yum -y install zlib-devel bzip2 bzip2-devel readline-devel sqlite sqlite-devel openssl-devel xz xz-devel libffi-devel

    创建目录

    mkdir -p /usr/local/python3

    下载安装包

    wget https://www.python.org/ftp/python/3.7.6/Python-3.7.6.tgz

    解压安装包

    tar -xvf Python-3.7.6.tgz

    编译安装Python

    cd Python-3.7.6 && ./configure --prefix=/usr/local/python3 && make && make install

    2、然后因为原操作系统中已经有老版本python，我们需要为其安装新版本python，就会遇到使用python3命令时可能会调用到老版本python的问题，有以下两个解决方案：

    1、直接创建Python3软链，利用命令 Python3 调用新版本python，与自带python不冲突；
    2、覆盖现有Python，使其指向新安装的Python，利用命令 python 便可调用新版本python；

    其中第二种方案详见CentOS7升级Python3版本 - 知乎 (zhihu.com)

    本文介绍第一种解决方案：

    在/usr/bin路径下创建python3软链，指向已安装的python3

    ln -s /usr/local/python3/bin/python3 /usr/bin/python3

    在/usr/bin路径下创建pip3软链，指向已安装的pip3

    ln -s /usr/local/python3/bin/pip3 /usr/bin/pip3

    配置完成之后，此时系统中存在两个python版本：命令python对应的仍是默认2.7版本；命令python3则对应新安装的3.6版本：

    [root@ZabbixServer]# python
    Python 2.7.5 (default, Aug  7 2019, 00:51:29)
    [GCC 4.8.5 20150623 (Red Hat 4.8.5-39)] on linux2
    Type "help", "copyright", "credits" or "license" for more information.

    >>>exit();
    [root@ZabbixServer]# python3
    Python 3.6.4 (default, Jun 15 2020, 16:17:18)
    [GCC 4.8.5 20150623 (Red Hat 4.8.5-39)] on linux
    Type "help", "copyright", "credits" or "license" for more information.
    >>>exit();
    [root@ZabbixServer]#

    当然，我们通过 python3 -V 和 pip3 -V也可以查看对应版本了：

    [root@ZabbixServer ~]# python3 -V
    Python 3.6.4
    [root@ZabbixServer ~]# pip3 -V
    pip 9.0.1 from /usr/local/python3/lib/python3.6/site-packages (python 3.6)
    [root@ZabbixServer ~]#

2、虚拟机设置代理

  • 方法1-通过自拟脚本设置shell的代理

    然后再给虚拟机设置代理路径

    $ cd /etc/profile.d  
$ vim proxy.sh

    内容编辑为：

    export proxy="http://192.168.11.1:7890"
export http_proxy=$proxy
export https_proxy=$proxy
export HTTP_PROXY=$proxy
export HTTPS_PROXY=$proxy
    其中，192.168.11.1替换为您的主机向虚拟机开放的ip，7890是主机的clash的代理端口，主机的所有要被代理的请求都是发送到该端口

    2、执行脚本文件

    sourch proxy.sh

    3、检查网络代理服务器是否成功

    curl www.google.com

    4、配置github网络代理

    git config --global http.https://github.com.proxy socks5://192.168.43.71:7890
  • 方法2-通过proxychains工具实现为任意指定程序挂代理
    其原理就是对指定程序的部分sockets相关API函数进行了hook，让程序的 sockets 数据走 SOCKS/HTTP 代理。

    先安装proxychains工具，在kali2022下直接输入以下命令即可：

    sudo apt-get install proxychains

    然后：

    sudo chmod a+rwx /etc/proxychains.conf
vim /etc/proxychains.conf

    最后一行：

    [ProxyList]
socks5  127.0.0.1 1086

    这里改为：

    [ProxyList]
http 192.168.11.1 7890
    这里192.168.11.1改为您的主机向虚拟机开放的IP（要选那个192.168开头的，意思是子网，然后要选那个.1结尾的，意思是虚拟路由器好像），且注意，这里只能设置一条规则不知道为啥

    这里之所以要填http，是因为clash和主机之间是使用http隧道传输技术进行传输

    然后就是使用，在shell中要使用某个应用程序时加上proxhchains4前缀即可，如下：

    $ proxychains4 curl www.google.com

    ↑然后这个例子顺便copy进去测试一下看看是否成功~

1、更新gcc

  • 以下方式适合centos7
    1. 确保你的系统已连接到Internet，并使用root权限登录。
    1. 运行以下命令来安装EPEL（Extra Packages for Enterprise Linux）存储库，它包含了很多社区维护的软件包，其中包括更新的GCC版本： yum install -y epel-release
    1. 添加devtoolset-9存储库，它包含GCC 9.x系列的新版本。运行以下命令： yum install -y centos-release-scl
    1. 安装GCC 9以及其他相关的工具和库。运行以下命令： yum install -y devtoolset-9-gcc devtoolset-9-gcc-c++ devtoolset-9-binutils
    1. 使用scl命令启用devtoolset-9环境变量。运行以下命令： scl enable devtoolset-9 bash
    1. 验证GCC版本是否已更新。运行以下命令： gcc --version 如果你看到输出显示GCC的版本号为9.x，则说明你已经成功地升级了GCC版本。
      2、保留该修改（设置环境变量）
    1. 打开shell配置文件，例如~/.bashrc或~/.profile： vi ~/.bashrc
    1. 在文件末尾添加以下行（假设您安装的gcc位于/opt/rh/devtoolset-9/root/usr/bin目录下）： export PATH=/opt/rh/devtoolset-9/root/usr/bin:$PATH
    1. 保存并关闭文件。
    1. 执行以下命令以使更改立即生效： source ~/.bashrc
如果上述操作还解决不了问题，建议是放弃centos7转投kali怀抱，以此一劳永逸
centos7对于gcc的配置很容易出问题，而且也不支持pwndbg，这也是我为什么很不推荐pwner使用sentos7的主要原因！

在win系统里面，只能是比较低版本的才能开telnet

3、为某个软件设置桌面快捷方式（可选）

【目录&文件】

按住ctrl+shift 然后用鼠标左键拖动到你的桌面目录文件即可（注意不是直接拖动到桌面）
同样你可以使用这种方式在任意地方创建该文件或目录的快捷方式。

【软件】

在图像界面下按照/usr/share/applications/这个路径打开文件目录，或者在终端运行$ nautilus /usr/share/applications/
这个文件夹下存放着已下载的各种软件的桌面快捷方式（后缀为.desktop），找到需要创建的快捷方式的软件，复制该文件到桌面，再赋予其执行权限即可。
