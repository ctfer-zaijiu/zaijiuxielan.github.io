gcc管理

1-查看当前的gcc版本和安装包

# 查看当前gcc环境
gcc --version

# 查看已有的gcc和g++包
# 一般默认是安装在/usr/bin里面的
ls /usr/bin/*gcc*
ls /usr/bin/*g++*

2-安装指定版本的gcc

1、选择需要的gcc版本

这里是各个版本glibc、gcc对应的发布时间，详情见：https://www.cnblogs.com/lightdb/p/6254046.html

2、安装gcc

  • 方法1-利用apt安装(推荐)

    我们可以用$ sudo apt-cache search <keyword>命令搜索看，会发现现在能支持直接下载的版本只有到gcc-11。

    去网上查了一下，是因为source.list源配置的版本号那一栏是最新的ubantu版本号

    可以将source.list源配置中的版本代号改为xenial，即16.04版

    vim /etc/apt/sources.list

# 写入
deb http://mirrors.aliyun.com/ubuntu/ xenial  main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ xenial-updates main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ xenial-security main restricted universe multiverse
deb http://mirrors.aliyun.com/ubuntu/ xenial-backports main restricted universe multiverse
# 退出
:wq

# 下载 更新源列表信息
sudo apt-get update

# 安装
apt-cache search gcc-5
apt-get install gcc-5
apt-get install g++-5

# 查看安装结果
dpkg -L gcc-5
# 可以看到安装在了：/usr/bin/gcc-5
  • 方法2-利用镜像站备份的源码安装

    下载源码

    （1）在gcc官网，根据其描述，导航到了一个英国的镜像FTP服务站：https://mirrorservice.org/sites/sourceware.org/pub/gcc/releases/gcc-6.5.0/

    （2）下载GCC6.5的tar.xz包

    # 解压
tar -xvf test.tar.xz

# 查看是否已安装
ls /usr/bin/gcc*

    注意，这里下载到的是源码！

3-设置“gcc”指令对应的程序

通过ubantu内置的工具update-alternatives来设置：

# 1、查看以前的优先级设置
sudo update-alternatives --config gcc

# 2、设置优先级
# 设置多个版本的gcc可供选择。80，60数字代表的是不同版本的优先级，默认最大数字版本的优先级。--slave: 这个选项允许你同时维护一组相关的符号链接，确保它们都指向同一组版本。例如，如果你切换 GCC 的版本，G++ 的版本也会相应改变。
ls /usr/bin/*gcc*
sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-5 80 --slave /usr/bin/g++ g++ /usr/bin/g++-5
sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-11 60 --slave /usr/bin/g++ g++ /usr/bin/g++-11

# 3、查看是否设置完成
gcc --version
