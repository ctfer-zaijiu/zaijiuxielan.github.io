【gadget】无需”binsh“，只要ret2libc即可一键getshell的one-gadget

【定义】

libc里面有现成的execve("/bin/sh",argv,envp)语句。获得libc基址，然后用onegadgets工具扫一下获得one-gadgets处的偏移然后ret2one-gadgets就getshell了。

它的优点如下：

1、可以避免遇到这个笔记里面system冗余检测的问题：

2、可以在我们没法填装binsh（一般是堆题或者格式化字符串）的时候使用

例如一些堆题做free@got覆盖后不方便填装binsh

3、一键getshell，使用方便，可以偷懒摆烂

缺点是onegadget不是随便用的，而是需要满足一些条件才能使用，onegadget工具会标出可以用的gadget和要用该gadget需满足的寄存器条件，如下：

$ one_gadget /lib/x86_64-linux-gnu/libc.so.6
constraints:
  rsp & 0xf == 0                     # 这个提示的意思就是在调用one_gadget前需要满足的条件
  rcx == NULL
  rbx == NULL || (u16)[rbx] == NULL
......

所以需要先劫持一些gadget来满足gadget的要求

但是有一个前辈的超级经验，就算不管是什么样的libc版本里，都有——read函数距离one_gadget的偏移恒为0x6109，将read的地址减去0x6109就可以使用one_gadget了

太感动了居然不用考虑libc版本/(ㄒoㄒ)/~~

【使用】

one_gadget工具搜索

one_gadget /lib/x86_64-linux-gnu/libc.so.6
