KEY机制

【定义】

为了防止同一个tcache被多次free，在libc2.27的高版本中，key机制被引入。

【结构】

1、新加入了key结构

原先的tcache定义如下：

typedef struct tcache_entry{
  struct tcache_entry *next;
} tcache_entry;

引入key机制后，其定义被更新为：

typedef struct tcache_entry{
  struct tcache_entry *next;
  struct tcache_perthread_struct *key; /* 新增指针 */
} tcache_entry;

这个新增的key域被用于doublefree检测

2、新加入了MAX_TCACHE_COUNT宏定义

#define MAX_TCACHE_COUNT 127 /* Maximum value of counts[] entries. */

其要求所有tcachebin的数目加起来最大数量为127

【机制】

1、限制tcache的最大数量

在do_set_tcache_count加入MAX_TCACHE_COUNT 的检测代码实现对tcachebin总数目的限制：

static inline int __always_inline do_set_tcache_count (size_t value){
  if (value <= MAX_TCACHE_COUNT){
      LIBC_PROBE (memory_tunable_tcache_count, 2, value, mp_.tcache_count);
      mp_.tcache_count = value;
    }
  return 1;
}

？这是不是把“劫持修改counts数组”的攻击方式给防出去了啊

2、一个针对doublefree的检测

tcache_put、tcache_get和int_free加入了一些新步骤

新加入的代码用黄色标出：
static __always_inline void tcache_put (mchunkptr chunk, size_t tc_idx){
  tcache_entry *e = (tcache_entry *) chunk2mem (chunk);
  assert (tc_idx < TCACHE_MAX_BINS);

  e->key = tcache; //将tcache_perthread_struct结构体的地址赋值给key

  e->next = tcache->entries[tc_idx];
  tcache->entries[tc_idx] = e;
  ++(tcache->counts[tc_idx]);
}
static __always_inline void *tcache_get (size_t tc_idx){
  tcache_entry *e = tcache->entries[tc_idx];
  assert (tc_idx < TCACHE_MAX_BINS);
  assert (tcache->entries[tc_idx] > 0);
  tcache->entries[tc_idx] = e->next;
  --(tcache->counts[tc_idx]);
  e->key = NULL; //清空e->key
  return (void *) e;
}
//int_free部分代码
#if USE_TCACHE{
	size_t tc_idx = csize2tidx (size);

	if (tcache != NULL && tc_idx < mp_.tcache_bins){
		//新增了这一段较为严格的doublefree检测机制，彻底防死了doublefree
		tcache_entry *e = (tcache_entry *) chunk2mem (p);
		if (__glibc_unlikely (e->key == tcache)){ // e->key检查是否为tcache_perthread_struct地址，这里是，用于防止doublefree
			tcache_entry *tmp;
			LIBC_PROBE (memory_tcache_double_free, 2, e, tc_idx);
			for (tmp = tcache->entries[tc_idx]; tmp; tmp = tmp->next) // 遍历整个当前大小的tcache链表，检查tcache中是否有一样的chunk
				if (tmp == e)
					malloc_printerr ("free(): double free detected in tcache 2");
		}

		if (tcache->counts[tc_idx] < mp_.tcache_count){
			tcache_put (p, tc_idx); //前面检查全部都过了以后，再调用这个tcache_put 来把tcache放进去
			return;
		}
	}
}
#endif

这里是先检查key，然后如果key出问题，再去遍历整个当前大小的tcache链表，检查tcache中是否有一样的chunk。

所以这两个检查我们只需要绕过去一个就好！
